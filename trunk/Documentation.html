Assemble

Assemble( string assemblycode, number address [, boolean writetoprocess] )

Description

Returns the assembled code.

Arguments

  1. string assemblycode
    Assembly instruction
  2. number address
    Address where the code will be relocated to.
  3. (optional) boolean writetoprocess
    If this is true it will also overwrite the memory in process at the given address.

Returns

  1. string data
  2. string error

Remarks

If the function fails data will be nil and the error is given.

RemoveInt3Breakpoint

RemoveInt3Breakpoint( number address [, number flags] )

Description

Removes a Int3 Breakpoint at the given address.

Arguments

  1. number address
    Where to remove the Breakpoint.
  2. (optional) number flags
    See the BP_ flags, for comfort the default flags are (BP_MANUAL | BP_SET | BP_BREAK) to remove a breakpoint.

Returns

  1. boolean success

Remarks

Passing the wrong flags may not remove the breakpoint, be careful what flags you use.

MEM_

  • MEM_COMMIT
    • Allocates memory charges (from the overall size of memory and the paging files on disk) for the specified reserved memory pages
  • MEM_RESERVE
    • Reserves a range of the process's virtual address space without allocating any actual physical storage in memory or in the paging file on disk
  • MEM_DECOMMIT
    • Decommits the specified region of committed pages. After the operation, the pages are in the reserved state
  • MEM_RELEASE
    • Releases the specified region of pages. After this operation, the pages are in the free state. If you specify this value, dwSize must be 0 (zero), and lpAddress must point to the base address returned by the RemoteAlloc function when the region is reserved
  • MEM_RESET
    • Indicates that data in the memory range specified by lpAddress and dwSize is no longer of interest
  • MEM_TOP_DOWN
    • Allocates memory at the highest possible address. This can be slower than regular allocations, especially when there are many allocations
  • MEM_WRITE_WATCH
    • Causes the system to track pages that are written to in the allocated region. If you specify this value, you must also specify MEM_RESERVE
  • MEM_PHYSICAL
    • Reserves an address range that can be used to map Address Windowing Extensions (AWE) pages
  • MEM_LARGE_PAGES
    • Allocates memory using large page support. The size and alignment must be a multiple of the large-page minimum
